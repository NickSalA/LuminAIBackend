# LuminAIBackend

feat: → Nueva funcionalidad (MINOR).

fix: → Corrección de bug (PATCH).

feat!: o BREAKING CHANGE: → Cambios incompatibles (MAJOR).

chore: → Tareas de mantenimiento (no cambian versión).

docs: → Documentación.

style: → Cambios de formato/estilo (no lógicos).

refactor: → Cambio interno que no altera funcionalidad.

test: → Añadir o modificar tests.

ci: → cambios en pipelines/CI/CD (como el workflow que subiste).

## Semantic Release

feat: y fix: → afectan versión automáticamente.

feat!: o BREAKING CHANGE: → salto de versión mayor.

ci:, docs:, chore:, etc. → no cambian versión, pero se documentan en el changelog.
